# Caden — Design System

**Always on. Always aware.**

Caden is a personal home AI — a Jarvis-like presence that lives on your own
hardware and runs every hour of every day. You speak to it; it listens, thinks,
and reaches you only when something is worth your attention. The primary way you
meet Caden is not an app full of panels — it is a **single calm canvas with one
input**, where Caden appears as a warm, living, morphing presence that drifts
across the space and gently steps aside to show you what you asked for.

This design system reskins and re-imagines the original Caden frontend into a
**warm, earthy, organic** visual language: clay and ochre and sage instead of
near-black and electric blue; soft shapes that morph and breathe instead of hard
rectangles; fluid, unhurried motion instead of sharp transitions.

## Quick start

Open `index.html` — the landing page with a live agent status view.

Open `ui_kits/canvas/index.html` — the interactive canvas: idle, listening,
thinking, and answer-reveal states, all wired up and demoable.

## File index

- `styles.css` — single entry point, link this.
- `tokens/` — design tokens (colors, type, spacing, motion, radius/shadow).
- `components/core/` — Button, Icon, IconButton, PresenceOrb, StatusDot.
- `ui_kits/canvas/` — the Caden Canvas interactive demo.
- `guidelines/` — type, colors, and brand specimen pages.
- `_ds_bundle.js` — pre-compiled component bundle (CDN React, no build step).
- `_ds_manifest.json` — design system manifest.
